# 0004: Dashboard filter styles use hardcoded colors

## Description

The new filter bar styles in [[src/styles/dashboard.css]] (lines 18-24) introduce three hardcoded hex color values that should use existing CSS custom properties from [[src/styles/variables.css]]. This is the third time in recent releases that new styles have bypassed the design token system.

## Affected Files

- [[src/styles/dashboard.css]]
- [[src/styles/variables.css]]

## Recommended Change

Replace hardcoded values with existing custom properties:

```css
/* Current */
.filter-bar {
  background: #f8fafc;
  border: 1px solid #e5e7eb;
}
.filter-bar__label {
  color: #1e293b;
}
.filter-bar__active {
  color: #3b82f6;
}

/* Recommended */
.filter-bar {
  background: var(--color-surface);
  border: 1px solid var(--color-border);
}
.filter-bar__label {
  color: var(--color-text-primary);
}
.filter-bar__active {
  color: var(--color-primary);
}
```

All four variables already exist in `variables.css`. No new tokens needed.

## Context

This continues a pattern flagged in [[0002]] (inline styles migration). The existing design token system is well-defined but not consistently adopted by new code. Consider adding a linting rule to flag hardcoded color values in CSS files. Related: [[0002]].
