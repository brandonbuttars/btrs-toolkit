# 0003: FilterBar $effect subscription missing cleanup

## Description

The `FilterBar` component in [[src/components/Dashboard/FilterBar.svelte]] creates a `$effect` on line 47 that subscribes to `filterStore.dateRange` but has no cleanup function. When the dashboard view is navigated away from and back to, each mount creates a new subscription without releasing the previous one, causing a memory leak.

## Affected Files

- [[src/components/Dashboard/FilterBar.svelte]]

## Recommended Change

Add a cleanup return to the `$effect`, or refactor to use `$derived` if the value is purely computed:

```svelte
// Current (leaks):
$effect(() => {
  const range = filterStore.dateRange;
  fetchDashboardData(range);
});

// Option 1 — cleanup:
$effect(() => {
  const range = filterStore.dateRange;
  const controller = new AbortController();
  fetchDashboardData(range, { signal: controller.signal });
  return () => controller.abort();
});

// Option 2 — if purely derived, use $derived instead:
const dashboardData = $derived(computeFromRange(filterStore.dateRange));
```

## Context

Discovered during release/1.2→1.3 analysis. Related: [[0001]] (date formatting utilities used inside this effect). The same subscription pattern appears in [[src/components/Dashboard/Dashboard.svelte]] but is handled correctly there — this component should follow the same pattern.
