# Obsidian Setup Guide

This guide walks through setting up Obsidian to browse the documents generated by the Code Review Toolkit.

## Prerequisites

- [Obsidian](https://obsidian.md) installed (free for personal use)
- A project that has been initialized with `/btrs-init-project`

## Opening your project as a vault

1. Open Obsidian
2. Click **"Open folder as vault"** (or **"Open"** → **"Open folder as vault"**)
3. Navigate to your project root directory and select it
4. Obsidian will detect the `.obsidian/` configuration that `/btrs-init-project` created

The first time you open a large project, Obsidian may take a moment to index files. The `/btrs-init-project` skill has already configured exclusions for `node_modules/`, `.git/`, build directories, etc., so only relevant files are indexed.

## Navigating generated documents

### File explorer

All generated documents live under `.local/` in the sidebar:

```
.local/
├── reviews/            ← MR review documents
├── releases/           ← Release notes (customer, engineering, tech debt)
├── tech-debt.md        ← Master backlog
└── tech-debt/          ← Individual tech debt items
```

### Wikilinks

Click any `[[wikilink]]` to navigate:

- In the tech debt master list, click `[[0003]]` to open that item's detail file
- In a review or tech debt item, click `[[src/components/Button.svelte]]` to open the actual source file
- Use `Ctrl/Cmd + Click` to open in a new pane for side-by-side viewing

### Tags

Open the **Tag pane** (right sidebar) to filter documents by:

- `tech-debt` — All tech debt items
- `priority/high`, `priority/medium`, `priority/low` — By urgency
- `area/ui`, `area/css`, `area/frontend`, `area/backend`, `area/api` — By subsystem
- `effort/small`, `effort/medium`, `effort/large` — By estimated time
- `release` — All release documentation
- `release/customer`, `release/engineering`, `release/tech-debt` — By audience
- `review` — All MR reviews
- `risk/low`, `risk/medium`, `risk/high` — By risk level

Click any tag to see all documents with that tag.

### Frontmatter properties

Each document has YAML frontmatter that Obsidian displays as a properties panel at the top. You can:

- Sort files by any property (date, priority, risk, etc.)
- Use the **Search** pane to filter by property values
- Build Bookmarks for common queries

### Callouts

Documents use Obsidian callouts for visual emphasis:

- `> [!tip]` — Low risk, positive findings (green)
- `> [!info]` — Suggestions, informational notes (blue)
- `> [!warning]` — Warnings, medium risk (yellow/orange)
- `> [!danger]` — High risk, breaking changes (red)
- `> [!failure]` — Critical findings (red)
- `> [!example]` — Deployment notes, examples (purple)
- `> [!note]` — General notes (gray)

## Recommended Obsidian plugins

These community plugins enhance the toolkit experience (all optional):

### Dataview

While the toolkit doesn't require Dataview, you can use it to create dynamic views:

```dataview
TABLE priority, effort, area, status
FROM #tech-debt AND -#index
WHERE status = "open"
SORT priority ASC
```

This creates a live table of all open tech debt items that updates automatically.

### Calendar

If you have the Calendar plugin, documents with `date` frontmatter appear on the calendar view, giving you a timeline of reviews and releases.

### Graph View

The built-in graph view (no plugin needed) shows relationships between documents. Tech debt items link to source files and to each other, reviews link to the tech debt items they created, and release summaries link to their backlog items.

Filter the graph to show only `.local/` files for a cleaner view of document relationships.

## Tips

- **Star important items** — Star high-priority tech debt items or the current release notes for quick access
- **Split panes** — Open the tech debt master list on one side and a detail file on the other
- **Quick switcher** — Use `Ctrl/Cmd + O` and type an item number (e.g., "0003") to jump directly to it
- **Backlinks** — Check the backlinks pane on any source file to see which reviews and tech debt items reference it
